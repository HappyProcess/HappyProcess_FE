현재 논의된 작업을 아래 순서로 진행한다:

1. **이슈 생성**: 작업 내용을 바탕으로 GitHub 이슈를 생성한다. (`gh issue create`)
2. **브랜치 생성**: `feature/issue-{N}-{짧은-설명}` 형식으로 브랜치를 만든다.
3. **플랜 작성 및 승인**: 코드 구현 계획을 플랜 모드로 작성하고 사용자 승인을 받는다.
4. **코드 구현**: 승인된 플랜대로 코드를 작성한다.
5. **커밋**: 변경 파일만 명시적으로 스테이징하고 한국어로 커밋한다.
6. **푸시**: `git push -u origin {브랜치명}`
7. **PR 생성**: `gh pr create` — body에 반드시 `Closes #{N}` 포함.

커밋과 PR에 AI 작성 흔적을 남기지 않는다.
